# Project Memory

Layer 1 of the memory system: concise, always-read project memory.

Keep this file short. Only stable facts, durable rules, and high-value preferences belong here. If a note is narrow, move it into `.workflow/memory/topics/`. If it is a settled tradeoff with rationale, record it in `.workflow/decisions/` instead.

Use this file as an index when possible:

- one-line stable fact here
- pointer to `.workflow/memory/topics/*.md` for detail
- pointer to `.workflow/decisions/*.md` when rationale matters
- pointer to `.workflow/memory/session-log.md` when the detail is merely recent, not durable

Example format:

- `[LEARN:claims] Do not state causality unless the identification assumptions are written explicitly.`
- `[LEARN:citations] Literature positioning must name the baseline comparison papers, not just the broad field.`
- `[RULE:scope] Keep the contribution claim narrower than the evidence base whenever identification is partial.`
